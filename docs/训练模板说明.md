# 训练模板说明

## 1. 训练模板解决什么问题

训练模板用于解决重复录入训练计划的问题。用户如果经常做固定的力量训练流程，例如“推日”“拉日”“腿日”，可以先保存为模板，后续创建训练记录时直接复用模板结构，而不是每次重新输入动作名称、组数目标和备注。

当前这一步只新增数据库结构，不改前端页面。下一步再实现 `/templates` 页面，用于创建、查看和套用训练模板。

## 2. 表关系说明

本次新增 3 张业务表：

1. `workout_templates`
   - 训练模板主表
   - 一条记录代表一个模板，例如“胸肩三头模板”

2. `workout_template_exercises`
   - 模板动作表
   - 通过 `template_id` 关联 `workout_templates.id`
   - 一条记录代表模板中的一个动作，例如“卧推”

3. `workout_template_sets`
   - 模板训练组表
   - 通过 `template_exercise_id` 关联 `workout_template_exercises.id`
   - 一条记录代表某个模板动作下的一组目标，例如“第 1 组 60kg x 8”

关系结构如下：

```text
workout_templates
  └─ workout_template_exercises
       └─ workout_template_sets
```

所有表都包含 `user_id`，并且都关联 `auth.users.id`。这意味着模板数据仍然由 Supabase Auth 用户体系管理，不会自建用户密码表，也不会保存 `password_hash`。

## 3. 如何执行 SQL

在 Supabase 控制台中执行以下步骤：

1. 打开项目
2. 进入 `SQL Editor`
3. 新建一个查询
4. 打开本地文件 `supabase/sql/003_add_workout_templates.sql`
5. 将文件内容粘贴到 SQL Editor
6. 点击执行

注意：

1. 不要在前端使用 `service_role` 或任何 secret key
2. 本次执行的是数据库迁移脚本，不是前端代码

## 4. 执行后如何检查表是否存在

执行成功后，在 Supabase 控制台中：

1. 打开 `Table Editor`
2. 查看 `public` schema
3. 确认以下表已经存在：
   - `workout_templates`
   - `workout_template_exercises`
   - `workout_template_sets`

同时可以检查字段是否齐全，例如：

1. `workout_templates.name`
2. `workout_template_exercises.exercise_name_snapshot`
3. `workout_template_sets.set_number`
4. `workout_template_sets.target_weight_kg`
5. `workout_template_sets.target_reps`

## 5. 如何确认 RLS 已开启

在 Supabase 控制台中：

1. 打开 `Authentication`，确认用户体系仍然使用 Supabase Auth
2. 打开 `Table Editor`
3. 依次进入：
   - `workout_templates`
   - `workout_template_exercises`
   - `workout_template_sets`
4. 检查每张表的 `RLS` 状态是否为启用
5. 检查每张表是否都存在 4 类策略：
   - `select`
   - `insert`
   - `update`
   - `delete`

本次 SQL 已写入以下规则：

1. `select` 使用 `user_id = auth.uid()`
2. `insert` 使用 `with check (user_id = auth.uid())`
3. `update` 同时使用 `using` 和 `with check`
4. `delete` 使用 `user_id = auth.uid()`

这表示用户只能访问自己的训练模板数据。

## 6. 迁移内容摘要

本次迁移额外包含以下内容：

1. 为 3 张表创建必要索引
2. 为 3 张表创建 `updated_at` 自动更新时间触发器
3. 如果数据库里还没有 `public.set_updated_at()` 函数，则自动创建
4. 为 3 张表开启行级访问控制
